## [I tried 5 Firebase alternatives](https://www.youtube.com/watch?v=SXmYUalHyYk)

### TL;DR
- Firebase is the industry leader for BaaS but its proprietary nature and unpredictable scaling costs are common concerns for developers.
- **Supabase** is a top choice for those who prefer PostgreSQL, offering an open-source, powerful SQL backend with Realtime features.
- **Appwrite** is another strong open-source alternative that can be easily self-hosted using Docker, providing a unified API for all backend needs.
- **AWS Amplify** provides deep integration with AWS services but has a steeper learning curve and can be complex to manage compared to other BaaS tools.
- **Nhost** focuses on a GraphQL-first approach, seamlessly combining PostgreSQL, Hasura, and cloud functions for a modern developer experience.

### Description Summary
Jeff Delaney from Fireship compares five popular Backend-as-a-Service (BaaS) platforms to Google's Firebase, including Supabase, Appwrite, AWS Amplify, Nhost, and MongoDB Realm. The video evaluates each alternative based on its features, pricing models, and how they solve common Firebase pain points like vendor lock-in and NoSQL limitations.

### Summary
The video addresses the common developer debate of whether to use Firebase or explore its modern competitors. It explains that while Firebase offers a seamless developer experience, developers often look elsewhere to avoid vendor lock-in and high, unpredictable costs. The video dives into five key alternatives:
1. **AWS Amplify**: The enterprise-focused AWS solution for building full-stack apps with mobile and web SDKs.
2. **Supabase**: Often called the "Open Source Firebase," it leverages PostgreSQL for a robust SQL backend and offers similar features like Auth, Realtime, and Functions.
3. **Appwrite**: A developer-centric platform that can be self-hosted on any server with Docker, providing a unified API for backend needs.
4. **Nhost**: A GraphQL-driven platform that integrates Hasura, PostgreSQL, and MinIO for file storage, aimed at speed and scalability.
5. **MongoDB Realm**: A mobile-first platform focusing on edge synchronization between a local database and a cloud-hosted MongoDB instance.
